Two pretty neat tricks with Java enumerations.

Lets say you have a simple enum that specifies directions that pieces move on a chessboard (for simplification, lets just consider major pieces without castling).

public enum Direction {
N, NNE, NE, ENE E, ESE, SE, SSE, S, SSW, SW, WSW, W, WNW, NW, NNW
}

Pretty standard right? Then you might have an interface such as Piece, with a method for returning the directions a piece can go...

public interface Piece {
public Direction[] getDirections() {
}

and have a class for each type of piece, with a method for each that returns individual

public enum Toy {
DOLL() {
@Override public void execute() {
System.out.println("I'm a doll.");
}
},
SOLDIER() {
@Override public void execute() {
System.out.println("I'm a soldier.");
}
};
//template method
public abstract void execute();
}

public class SwitchEnum {

private enum Day {
MON, TUE, WED, THU, FRI, SAT, SUN
}

public static String response(final String s) {
switch(Day.valueOf(s)) {
case FRI:
return "almost...";
case SAT:
case SUN:
return "yay!";
default:
return "zzz";
}
}

public static void main(String[] args) {
System.out.println("Switching...");

String day;

day = "MON";
System.out.println(day + "t--> " + response(day));
day = "TUE";
System.out.println(day + "t--> " + response(day));
day = "WED";
System.out.println(day + "t--> " + response(day));
day = "FRI";
System.out.println(day + "t--> " + response(day));
day = "SAT";
System.out.println(day + "t--> " + response(day));

}

}
