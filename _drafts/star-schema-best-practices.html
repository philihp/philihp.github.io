Here are some tips that have helped me out.

Suffix fact tables with _fact
Suffix dimension tables with _dim

Having a fact and dim table with the same name is probably code-smell, so make sure you have a good reason.

Name dimension IDs as the table name + "_id", for example "customer_id" for a customer_dim table.

Use the same name of the dimension_id when possible (not always possible, since you might have "bill_address_id" and "ship_address_id" and an address_dim table with just "address_id".)

Include a "count" field in the fact table with "1" in every row. This comes in very convenient for summations.
